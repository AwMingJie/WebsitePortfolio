01 / Challenge

Build for scale and operational visibility.

The project required a cloud architecture that could scale the web workload, expose operational health, and support serverless application workflows around a .NET Core 6.0 web application.

02 / My role and scope

From architecture through validation.

I designed and deployed the architecture, configured monitoring and alerts, developed the serverless workflows, documented architectural decisions, and conducted failover and load testing.

03 / Implementation

A monitored, event-driven AWS workload.

  • Hosted the application on EC2 within a VPC, using an Elastic Load Balancer and Auto Scaling Groups with CPU scaling policies.
  • Built CloudWatch dashboards and SNS alerts for CPU utilisation, network traffic, and instance health.
  • Used Lambda, API Gateway, and DynamoDB to capture and retrieve product records through RESTful endpoints.
  • Used an S3-triggered Lambda function to apply watermarks to uploaded images.
  • Applied AWS Well-Architected Framework principles when documenting architecture decisions.

04 / Validation and outcome

Test the design, not just the deployment.

The architecture was exercised through failover and load testing. The completed deployment combined scalable web hosting, operational monitoring and alerting, a serverless product-data API, and automated image processing.

Evidence boundary

This public summary intentionally omits account identifiers and environment-specific configuration. No performance figures are published because verified test measurements were not supplied with the portfolio material.

05 / What it demonstrates

Resilience requires visibility and deliberate testing.

The project demonstrates how high availability, operational monitoring, event-driven automation, and documented decisions work together. Cloud resilience depends on deliberate failure testing, not only on deploying infrastructure.
